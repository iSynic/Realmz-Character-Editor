# Realmz Character Editor

A browser-only React/TypeScript editor for loose Realmz character files.

The app accepts a valid 872-byte Realmz character file, keeps all bytes local in
the browser, validates editable fields, and downloads a modified 872-byte file.
There is no backend and no runtime dependency on a local Realmz install.

## Current Status

- Static Vite app with React, TypeScript, Vitest, Playwright, and Oxlint.
- Binary parser/serializer for the packed Realmz character record.
- Editor tabs for core stats, spells, conditions, abilities, items, and appearance.
- Generated Realmz metadata for item, race, caste, spell, condition, and icon IDs.
- Placeholder generated SVG art is checked in for now; real Realmz art extraction is
  the next asset step.

## Development

```powershell
npm ci
npm run dev
```

Useful checks:

```powershell
npm run lint
npm test
npm run build
npm run test:e2e
```

The full fixture tests use a local Realmz checkout. By default they look for
`F:\Realmz`; set `REALMZ_ROOT` to override that path. Tests that require local
Realmz character fixtures skip cleanly when the fixture directory is unavailable.

## Metadata And Assets

Realmz metadata and current placeholder assets are generated by:

```powershell
npm run generate:metadata
```

The extractor defaults to `F:\Realmz` and can be pointed elsewhere with
`REALMZ_ROOT`. Generated output is checked in under `src/generated` and
`public/realmz-assets` so the built editor remains static.

## GitHub Pages

The app is deployed as a static GitHub Pages site at:

```text
https://isynic.github.io/Realmz-Character-Editor/
```

Repository Settings -> Pages should use `GitHub Actions` as the build and
deployment source. The Vite `base` setting is configured for the project Pages
path above; if the site moves to a custom domain, change `base` back to `/`.
